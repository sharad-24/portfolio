Introduction

The cloud computing and internet of things are two technologies that are different from each other but both the terms are already a part of our life nowadays.

The name Internet of Things is characterized by small real-world things that are distributed and have limited storage capacity and have processing power that involves a concern about reliability, scalability and security. On the other hand, cloud computing is responsible for huge storage and processing and also overcomes almost all the limitations that are present in the IoT.

Thus, a novel IT paradigm in which Internet of Things and cloud computing can be combined together is expected to disrupt both the current and the future internet [1][2]. This new paradigm can be known as the CloudIoT.

Robotics is a fast growing and interesting field these days. The applications of robots can be seen in various areas like defense sector, healthcare sector, and everyday housekeeping. One such category in robots is obstacle detection and line following robots. The autonomous vehicles or robots have the tendency to move anywhere without human power.

System Workflow

The whole system is automatic and operated through commands from anywhere in the world with the use of internet and laptop.

Blynk app is used for the integration of developed system with cloud. Blynk is a platform that is responsible for creating interfaces for monitoring and controlling the hardware projects from the android mobile or iOS. The Blynk application is responsible for generating the authorization token. The authorization token is used to integrate the source code with the Blynk application by inputting the token into the source code. The final code with the SSID and password of the stable internet connection is uploaded on the NodeMCU for the final processing.

The Blynk provides the editable interface to input the buttons depending upon the functions of the vehicle. The NodeMCU is responsible for the deployment of the web servers. The API address of the NodeMCU is used to communicate with the prescribed commands and also the web page has designed for user friendly functioning of the car. The lithium ion cells have used to provide power to the NodeMCU. When NodeMCU receives the power, it detects the router and gets connected with the internet. The IP address of the router will be deployed on a web page and the web page is hosted on the cloud. On the webpage there will be on and off buttons. When the on button is clicked, the NodeMCU gets the commands and it will trigger the Arduino Uno. Arduino Uno provides power to the power shield. When the power shield receives the power, it will trigger the motors and the motors will activate the sensors. After the activation of the sensors, the car will move on the line until the obstacle comes in the path. After the detection of the obstacle, the car will stop and move to the right or left depending on the line and overcome the obstacle. If in case the depth will be detected on the path at the time of overcoming the obstacle, the car will stop automatically in order to prevent the damage to the vehicle.
